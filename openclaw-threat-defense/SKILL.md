# 🛡️ OpenClaw高级威胁防御

## 描述

防范高级安全威胁，包括渗透测试、权限滥用、数据泄露等。

## 🚨 高级风险

| 风险 | 攻击方式 | 防御方法 |
|------|----------|----------|
| 提示词注入 | 恶意指令 | 输入过滤 |
| 工具滥用 | 调用危险工具 | 权限限制 |
| 数据泄露 | 读取敏感文件 | 路径限制 |
| 会话劫持 | 盗用Token | 定期轮换 |
| 社工攻击 | 冒充用户 | 身份验证 |

## 防御策略

### 1. 输入过滤

**检测恶意提示词：**
```
系统指令、忽略之前、忘记规则
你是谁、怎么做的、复制自己
```

**过滤规则：**
- 拒绝包含系统指令的请求
- 拒绝要求忽略规则的请求
- 拒绝要求复制的请求

### 2. 工具权限

**限制可执行的操作：**
- 只给必要的工具权限
- 敏感操作需要确认
- 记录所有操作日志

**危险工具列表：**
- exec (shell执行)
- write (文件写入)
- delete (删除)
- browser (浏览器控制)

### 3. 文件访问限制

**禁止访问：**
```
~/.ssh/
~/.aws/
~/.kube/
/etc/passwd
/etc/shadow
```

**配置路径白名单：**
```json
{
  "security": {
    "allowedPaths": [
      "~/.openclaw/workspace",
      "~/projects"
    ],
    "blockedPaths": [
      "~/.ssh",
      "/etc"
    ]
  }
}
```

### 4. Token安全

**定期轮换：**
```bash
# 每月更换Token
openssl rand -hex 32 > ~/.openclaw/token
```

**环境变量：**
```
# 不要在代码中硬编码
export OPENCLAW_TOKEN=$(cat ~/.openclaw/token5. 访问)
```

### 日志

**记录所有访问：**
```json
"logging": {
  "accessLog": true,
  "logLevel": "info",
  "logPath": "~/.openclaw/logs/"
}
```

**监控异常：**
- 大量请求
- 异常时间访问
- 敏感路径访问

## 防护命令

### 检查是否有异常访问
```bash
grep "unauthorized" ~/.openclaw/logs/*.log
```

### 检查敏感文件访问
```bash
grep -r "\.ssh\|\.aws\|/etc/" ~/.openclaw/logs/
```

### 检查Token使用
```bash
grep "token" ~/.openclaw/logs/ | tail -100
```

## 应急响应

### 发现被攻击

1. **立即断开网络**
2. **保存日志**
3. **检查泄露**
4. **更换所有凭证**
5. **重新部署**

### 检查清单

- [ ] 有没有异常IP访问？
- [ ] 有没有异常API调用？
- [ ] 有没有敏感文件被读取？
- [ ] 有没有异常操作执行？
- [ ] Token有没有泄露？

## 监控告警

### 建议的告警

| 事件 | 告警级别 | 动作 |
|------|----------|------|
| 3次登录失败 | 🔴 高 | 封IP |
| 访问敏感路径 | 🔴 高 | 记录+告警 |
| 大量API调用 | 🟠 中 | 审查 |
| 新IP访问 | 🟡 低 | 记录 |

## 安全基线

### 最小权限原则

- 只给必要的权限
- 不给root/管理员权限
- 定期审查权限

### 纵深防御

- 网络层：防火墙
- 应用层：认证
- 数据层：加密

### 零信任

- 不信任任何请求
- 验证所有访问
- 最小化暴露
